# Claude Settings Reference

This file preserves Claude-origin settings and local permission context as
reference material only. It does not translate these settings into Codex runtime
configuration.

No `.codex/config.toml` is generated by this migration. If future maintainers add
one, keep it minimal and do not enable executable hooks, destructive sandbox
settings, approval overrides, or stale model pins without a separate plan.

## Source: `.claude/settings.json`

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": ".claude/hooks/check-expert-update.sh"
          }
        ]
      }
    ]
  }
}
```
